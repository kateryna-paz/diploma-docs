# 6. Вимоги до якості

## 6.1 Метрики якості моделі

Для оцінки якості роботи системи використовуються стандартні метрики класифікації:

- **Precision** — частка реальних аномалій серед усіх виявлених: `TP / (TP + FP)`
- **Recall** — частка виявлених аномалій серед усіх реальних: `TP / (TP + FN)`
- **F1-score** — гармонічне середнє Precision і Recall: `2 × P × R / (P + R)`
  Цільові значення на тестовому датасеті CICIDS2017:

| Метрика             | Ціль   |
| ------------------- | ------ |
| F1-score            | ≥ 0.85 |
| Precision           | ≥ 0.80 |
| Recall              | ≥ 0.85 |
| False Positive Rate | ≤ 10%  |

## 6.3 Критерії приймання

Система вважається готовою, якщо:

1. На тестовому датасеті досягнуто F1-score ≥ 0.85.
2. Рівень хибних спрацювань на нормальному трафіку не перевищує 10%.
3. Кожна виявлена аномалія супроводжується SHAP-поясненням.
4. Система запускається і починає аналіз без додаткового налаштування після команди `docker compose up`.

# 4. Вимоги до складу та архітектури

## 4.1 Загальна архітектура системи

Система складається з чотирьох послідовних шарів.
Спочатку дані надходять з джерел: метрики у форматі Prometheus, access logs Nginx/Apache та структуровані JSON-логи застосунку. Модуль збору даних зчитує їх і передає далі у вигляді єдиного потоку подій.

Потім препроцесор формує вікна ознак та нормалізує числові значення. Підготовлені вектори одночасно потрапляють до двох детекторів: Isolation Forest обробляє поточні числові метрики, а LSTM-автоенкодер аналізує часові послідовності запитів. Результати обох детекторів об'єднуються у єдиний показник аномальності.

Якщо показник перевищує поріг, останній шар формує SHAP-пояснення, зберігає аномалію та відправляє сповіщення через webhook і REST API.

---

## 4.2 Компоненти системи

### 4.2.1 Модуль збору даних (`collector`)

- **Вхід:** Prometheus scrape endpoint, Filebeat TCP stream, HTTP Push API.
- **Вихід:** Нормалізований потік подій у внутрішній черзі (Redis Streams або in-memory queue).
- **Вимоги:** буферизація до 10 000 подій при деградації детектора.

### 4.2.2 Препроцесор (`preprocessor`)

- **Відповідальність:** формування векторів ознак, STL-декомпозування, нормалізація.
- **Конфігурація:** розмір вікна, список ознак, метод нормалізації — через YAML-конфіг.
- **Артефакти:** збережені параметри нормалізації (scaler.pkl) разом з моделлю.

### 4.2.3 Детектор аномалій (`detector`)

- **iForest-компонент:** scikit-learn `IsolationForest`, параметри: `n_estimators=200`, `contamination=auto`.
- **LSTM-AE компонент:** PyTorch LSTM encoder–decoder, розмір прихованого шару 64, кількість шарів 2.
- **Score fusion:** зважене середнє з вагами, що оновлюються за ретроспективним feedback.

### 4.2.4 SHAP-пояснювач (`explainer`)

- Бібліотека: `shap` (TreeExplainer для iForest, GradientExplainer для LSTM-AE).
- Формат виходу: JSON з топ-5 ознаками та їхніми SHAP-значеннями.

### 4.2.5 REST API (`api-server`)

- Фреймворк: FastAPI (Python).
- Документація: автоматично генерується OpenAPI 3.0 специфікація.
- Аутентифікація: JWT Bearer.

## 4.3 Зовнішні інтерфейси

| Інтерфейс                | Протокол            | Опис                          |
| ------------------------ | ------------------- | ----------------------------- |
| Prometheus metrics input | HTTP GET `/metrics` | Pull-модель збору метрик      |
| Log ingestion            | TCP JSON            | Прийом структурованих логів   |
| REST API                 | HTTP/1.1 + JSON     | Запит статусу, аномалій, SHAP |
| Webhook output           | HTTP POST JSON      | Сповіщення про аномалії       |
| Grafana datasource       | HTTP JSON           | Виведення аномалій на дашборд |

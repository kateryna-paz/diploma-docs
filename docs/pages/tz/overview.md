# Технічне завдання

## Система виявлення аномалій у web-сервісах на основі машинного навчання

**Позначення документа:** ТЗ-2025-ADS-01  
**Версія:** 1.0  
**Статус:** Затверджено  
**Дата:** 2026

---

Цей розділ містить повне технічне завдання (ТЗ) на розробку програмного продукту — системи автоматичного виявлення аномалій у web-сервісах із застосуванням методів машинного навчання.

Документ розроблено відповідно до вимог [SRS (Software Requirements Specification)](https://www.geeksforgeeks.org/software-requirement-specification-srs-format/) та рекомендацій [стандартів ТЗ на розробку ПЗ](https://habr.com/ru/articles/328822/).

## Структура ТЗ

| Розділ                                 | Зміст                                         |
| -------------------------------------- | --------------------------------------------- |
| [1. Загальні відомості](./general)     | Найменування, замовник, підстава для розробки |
| [2. Призначення та мета](./purpose)    | Функціональне та експлуатаційне призначення   |
| [3. Вимоги до системи](./requirements) | Функціональні та нефункціональні вимоги       |
| [4. Вимоги до складу](./components)    | Архітектура, модулі, інтерфейси               |
| [5. Вимоги до даних](./data)           | Вхідні/вихідні дані, формати, датасети        |
| [6. Вимоги до якості](./quality)       | Метрики якості ML-моделей, надійність         |
| [7. Стадії розробки](./stages)         | Етапи, терміни, результати                    |

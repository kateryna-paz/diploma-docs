# 2. Призначення та мета розробки

## 2.1 Функціональне призначення

Система ADS призначена для **автоматичного виявлення аномальної поведінки у web-сервісах** в режимі реального часу шляхом аналізу:

- HTTP access logs (методи, ендпоінти, коди відповідей, розміри тіл);
- метрик продуктивності (latency p50/p90/p99, error rate, throughput, RPS);
- системних показників (CPU, RAM, кількість потоків, GC-паузи);
- часових патернів трафіку з урахуванням сезонності.

Система орієнтована на три класи аномалій:

| Тип аномалії | Опис | Приклад |
|---|---|---|
| **Точкова** | Одиничне відхилення окремої метрики | Latency 47 с при нормі 0.3 с |
| **Контекстуальна** | Нормальне значення у нетиповому контексті | 200 RPS о 3:00 ночі |
| **Колективна** | Підозріла послідовність нормальних запитів | Reconnaissance → SQL injection → file read |

---

## 2.2 Експлуатаційне призначення

Система призначена для використання у таких сценаріях:

- **Операційний моніторинг** — виявлення деградації сервісів, витоків пам'яті, перевантаження черг.
- **Виявлення кіберзагроз** — DDoS-атаки, SQL-ін'єкції, brute force, slowloris, account takeover.
- **Аналіз після інциденту** — ретроспективний пошук аномалій у збережених логах.

**Цільова аудиторія:**

- DevOps / SRE інженери, що обслуговують production web-сервіси.
- Аналітики Security Operations Center (SOC).
- Дослідники у галузі машинного навчання та кібербезпеки.

---

## 2.3 Мета розробки

Мета роботи — дослідити та реалізувати систему, яка:

1. Перевершує точність порогових детекторів (Prometheus alerts) за рахунок ML.
2. Пояснює кожне спрацювання через SHAP-аналіз ознак.
3. Адаптується до drift трафіку без ручного перелаштування порогів.
4. Масштабується до потоків понад 50 000 подій/хвилину.

---

## 2.4 Очікуваний ефект

- Скорочення Mean Time to Detect (MTTD) аномалій у порівнянні з пороговими системами.
- Зменшення кількості хибних спрацювань (False Positive Rate) порівняно з Isolation Forest без ансамблю.
- Забезпечення інтерпретованості рішень для аналітика SOC.

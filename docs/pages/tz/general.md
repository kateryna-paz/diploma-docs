# 1. Загальні відомості

## 1.1 Найменування системи

**Повна назва:** Система автоматичного виявлення аномалій у web-сервісах на основі методів машинного навчання

**Скорочена назва:** ADS (Anomaly Detection System)

**Позначення:** АДС-2026

## 1.3 Підстава для розробки

Підставою для розробки є:

- Завдання на виконання кваліфікаційної (магістерської) роботи за спеціальністю **122 «Комп'ютерні науки»**.
- Актуальність задачі автоматичного виявлення аномалій в умовах переходу бізнес-критичних сервісів до хмарних і мікросервісних архітектур.
- Недостатність існуючих відкритих рішень для комплексного виявлення точкових, контекстуальних та колективних аномалій у real-time режимі.

## 1.5 Джерела розробки

Технічне завдання розроблено на основі:

1. Аналізу наукової літератури з виявлення аномалій (Chandola et al., 2009; Liu et al., 2008).
2. Огляду існуючих систем моніторингу: Prometheus, Datadog, Dynatrace, Elastic.
3. Вимог стандарту SRS (IEEE 830-1998 / ISO/IEC 29148:2018).
4. Рекомендацій [habr.com: Стандарти та шаблони для ТЗ](https://habr.com/ru/articles/328822/).
